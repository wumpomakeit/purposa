How it works

From noisy proposal
to clear verdict in seconds

No reading forums. No second-guessing delegates. Just paste the link and get a verdict you can stand behind.

1

Paste the URL

Drop any Snapshot proposal link into the API or interface. Purposa pulls the full proposal body, vote tally, quorum data, and discussion links automatically.

2

Four agents read it

Summarizer, Critic, Risk Assessor, and Judge each analyze the proposal independently and in parallel. No single model decides alone.

3

Get a verdict

A recommendation (For / Against / Abstain), confidence score, risk flags, pros & cons, and a full reasoning trace — every agent's logic visible and auditable.

4

Pay per analysis

Each analysis is billed exactly via x402 using your OKX Agentic Wallet. No subscription. No idle charges. You pay for what you use, settled in stablecoin.

5

Confirm & vote

Agree with the verdict or override it — your choice. Confirm once. Purposa signs and submits the EIP-712 vote to Snapshot Hub on your behalf. Always requires your explicit approval.

What you get

Every analysis.
Every time.

Not a summarizer that tells you what a proposal says. A cross-checked panel that tells you whether you should vote for it — and exactly why.

🤝

Multi-Agent Cross-Check

Four models with different mandates analyze the same proposal independently. The Judge reconciles disagreements rather than averaging them — you see where agents differed and why the verdict landed where it did.

🔍

Auditable Reasoning Trace

Every verdict is paired with a trace ID. Open it and read exactly what each agent said, what it flagged, and how the Judge resolved it. No black box verdicts — every recommendation is evidence-backed and reviewable.

⚑

Risk Flag Detection

The Risk Assessor looks for treasury accountability gaps, insufficient discussion periods, quorum failure risk, vague scope, governance concentration, and proposer conflict of interest — and rates each flag by severity.

✅

One-Click Vote Execution

Once you've reviewed the verdict, confirm your choice. Purposa signs the EIP-712 vote message inside the OKX Agentic Wallet's TEE and submits it to Snapshot Hub. The private key never leaves the enclave.

⚡

Pay-Per-Call Pricing

Analysis is billed per proposal via x402 — the open agent payment protocol. Exact-scheme payment, settled in USD₮0 on X Layer. No subscription tier, no monthly commitment. Use it when you need it.

Common questions

Things worth
knowing upfront

Currently, Purposa works with any active or closed proposal on Snapshot (snapshot.org). This covers most major DAOs — Uniswap, ENS, Arbitrum, Aave, Compound, Balancer, and hundreds of others that use Snapshot for off-chain governance.

On-chain governance platforms (Tally, Compound Governor Bravo, OpenZeppelin Governor) are the highest-priority v2 item. They're not supported yet because each requires handling different contract standards and gas estimation — complexity that doesn't belong in an MVP.

Each analysis is billed pay-per-call via the x402 protocol (exact scheme). The cost is fixed per call, deducted from your OKX Agentic Wallet in USD₮0 on X Layer. There is no subscription, no monthly fee, and no charge for browsing.

The payment amount is shown in the HTTP 402 response before you commit — you can inspect it, confirm, and pay using onchainos payment pay. Vote submission is not separately billed.

Purposa is a research tool that hands you a verdict. What you do with that verdict is entirely your decision. Autonomous voting without human approval is something Purposa will not do in any version.

Purposa provides analysis and a recommendation — not a guarantee. The confidence score reflects agreement between agents, not certainty about the outcome. Governance results depend on political context, delegation dynamics, and late-breaking information that no model can fully anticipate.

Always read the reasoning trace before voting, especially on high-stakes proposals. Purposa is a research assistant, not a fiduciary. You remain responsible for your votes.

Currently, Purposa integrates with the OKX Agentic Wallet via the onchainos CLI for both payment and vote submission. The Agentic Wallet's private key is generated and stored inside a TEE — you authenticate via API key or email, and the key never leaves the enclave.

Support for other EIP-712-compatible wallets is planned but not yet implemented. For now, an OKX Agentic Wallet is required.

Proposal text is fetched from Snapshot's public GraphQL API and passed through the LLM pipeline. Purposa stores the analysis result — summary, verdict, reasoning trace — keyed to a trace ID. No personally identifying information is stored.

Your wallet address is used only at vote submission time and is not logged against the analysis. Trace files are stored locally by default and are not shared with third parties.

It's the clearest next step and the highest-priority v2 item, but there's no firm date. On-chain governance requires handling multiple contract standards (Governor Bravo, OZ Governor, etc.), gas estimation, and transaction submission — complexity that's out of scope for an MVP that needs to demonstrate the core value proposition first.

Snapshot currently covers the majority of active DAO governance volume, including Uniswap, ENS, Arbitrum, and Aave — so the MVP is not a thin edge case.

Privacy Policy

Last updated: July 2026 · Version 0.1 (draft)

What we collect

Purposa fetches proposal data from Snapshot's public GraphQL API on your behalf. We store the analysis result (summary, verdict, reasoning trace) indexed by a randomly generated trace ID. We do not collect your name, email, or any personally identifying information.

Wallet addresses

Your wallet address is used only to submit votes to Snapshot Hub when you explicitly confirm a vote. It is not stored alongside analysis records or used for any other purpose.

LLM providers

Proposal text is passed to NVIDIA NIM (hosted LLM API) for analysis. NVIDIA's data handling policies apply to data transmitted to their API. We do not send wallet addresses or personal information to LLM providers.

Trace storage

Analysis traces are stored locally on the server by default. They contain the proposal ID, the verdict, and agent reasoning. They do not contain your wallet address or payment information.

Cookies & tracking

Purposa does not use cookies, analytics trackers, or third-party tracking scripts on this page.

Contact

For privacy questions, open an issue on the GitHub repository.
